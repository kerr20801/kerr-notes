# 自建 LLM 不是無敵，是另一種工具

> **結論先說**：Ollama 跑起來不等於你有了 GPT-4 等級的能力。地端 LLM 是一個在資料邊界內、延遲可控、成本固定的工具，不是雲端 LLM 的平替版。搞清楚這個，才知道它應該放在架構的哪個位置。

---

## 「我們自建了 LLM」這句話背後的誤解

很多人裝完 Ollama、跑起來 llama3 或 qwen，第一個感覺是：

> 「我們有地端 AI 了，資料不出去，還免費，這樣不是很好？」

這個感覺對了一半。

對的部分：資料確實不出去，推論成本確實固定。

錯的部分：**「有地端 AI」不等於「有跟雲端一樣的能力」**。這兩件事是完全不同的。

---

## 現實數字

| 項目 | 雲端（GPT / Claude Sonnet） | 地端（7B~13B） | 地端（70B，需高階 GPU）|
|------|-------------------------------|---------------|----------------------|
| 推理能力 | 基準 100% | 約 15~25% | 約 50~60% |
| 上下文長度 | 128K~200K token | 4K~32K | 32K~128K |
| 多步推理 | 強 | 容易繞圈、漏步 | 尚可 |
| 程式生成準確率 | 高 | 中下 | 中 |
| 需要硬體 | 無 | 普通 CPU/GPU 可跑 | 需要 A100/H100 等級 |
| 每月成本 | 依用量（可能貴） | 固定（電費+硬體折舊）| 固定但硬體成本高 |

7B 模型跑在本機，能力大約是雲端頂級模型的五分之一到四分之一。這不是在貶低地端 LLM，是實際的能力分布。

---

## 地端 LLM 真正適合的任務

不是「什麼都能做」，是這些：

**適合地端**
- 固定格式的結構化輸出（JSON 抽取、分類標注）
- 重複性高、prompt 結構固定的任務
- 資料絕對不能出邊界的場景（法規要求、機密資料）
- 延遲敏感、需要毫秒級回應的即時任務
- 呼叫量極大、雲端費用不划算的場景

**不適合地端**
- 複雜多步推理（鏈式思考、需要反覆驗證）
- 開放式分析（需要廣泛世界知識）
- 程式架構設計、複雜 debug
- 任何需要「真的理解語意」而不只是「匹配 pattern」的任務

---

## 錯誤架構長這樣

```
所有任務
    ↓
地端 LLM（因為「資料不出去比較安全」）
    ↓
推理品質差 → 結果不準 → 不信任 AI → 放棄
```

這不是地端 LLM 不好，是把它放在它做不好的位置上。

---

## 正確的混合架構邏輯

不是「地端 vs 雲端」二選一，是**按任務分流**：

```
任務進來
    ↓
這個任務的資料能出邊界嗎？
    ├── 不能 → 地端 LLM（接受能力限制）or 純規則/ML（不用 LLM）
    └── 能 → 這個任務需要複雜推理嗎？
                ├── 需要 → 雲端 LLM
                └── 不需要（固定格式、重複高）→ 地端 LLM 或 ML 模型
```

「資料能不能出去」是硬條件，不是選項。「能力夠不夠」是在硬條件通過後才考慮的問題。

## 一個實際設計的例子

以檔案安全掃描為例（nas-ai 類型的系統）：

| 判斷項目 | 用什麼 | 為什麼 |
|---------|--------|--------|
| 副檔名黑名單 | 規則 | 不需要 LLM，一個 list 搞定 |
| MIME 偵測 | libmagic | 確定性問題，規則比模型準 |
| 異常 entropy | 統計算法 | 數學，不是語意 |
| 異常行為偵測 | Isolation Forest | ML 夠用，不需要 LLM |
| DLP 敏感資料 | regex + 規則引擎 | pattern matching，規則清楚 |
| 告警摘要說明 | 地端 LLM（可選）| 只做文字生成，不做判斷 |

這個系統完全不依賴 LLM 做核心決策。LLM 如果有，只是在最後一步生成人讀的說明。把 LLM 拿掉，系統照樣跑，準確率不變。

這才是地端 LLM 的正確定位：**輔助輸出，不是核心判斷。**

---

## 為什麼這個誤解這麼常見

因為 Ollama 太好裝了。

十年前要跑一個 ML 模型，要設定 CUDA、裝 framework、處理 dependency hell，門檻夠高，逼著人先想清楚「這東西能做什麼」。

現在 `ollama pull llama3 && ollama run llama3`，三分鐘跑起來，感覺很強，就開始接進系統。

**工具越容易用，越需要想清楚「用在哪裡」。**

---

## 一句話版本

> 地端 LLM 解決的問題是**資料主權和成本**，不是**能力**。如果你的問題是「我需要更強的推理能力」，地端 LLM 不是答案。如果你的問題是「這個資料不能出去但我需要 AI 處理」，那才是它的場。
