💥 避坑指南：為什麼 12GB 顯存跑不動 14GB 的大模型？

在規劃本地端 AI 推理伺服器時，最常出現的一個迷思是：「我買了 RTX 4070 12GB 或 RTX 3060 12GB，是不是就能跑滿 12GB 甚至 14GB 的模型？」

答案是：純 GPU 運算下不行。但透過 CPU 協同作戰可以，只是速度會顯著下降。

本文將以目前最多人使用的 Ollama 為例，並針對三張主流顯卡（RTX 3060 12G、RTX 3060 Ti 8G、RTX 4070 12G），解析本地部署 LLM 時的「顯存陷阱」、效能真相，以及如何利用 CPU 混搭來「越級打怪」。

📊 顯存的三分天下鐵則

很多人算 VRAM 只看 .gguf 檔案有多大，但實際上 Ollama 運行時的顯存消耗公式如下：

Total VRAM = 📦 模型權重 (Weights) + 🧠 KV Cache (上下文) + ⚙️ 框架開銷 (Overhead)

📦 模型權重 (Weights)： 這是靜態的。例如一個 14B 的 Q4 模型，光是載入就會吃掉約 8.5GB。

🧠 KV Cache： 這是動態的「吃 RAM 怪獸」。為了不重複計算過去的 token，模型必須把 Context（對話紀錄/長文章）暫存起來。上下文越長，吃掉的 VRAM 就越多（通常需要 1GB ~ 2GB 以上）。

⚙️ 框架開銷 (Overhead)： Ollama (基於 llama.cpp) 啟動 CUDA 與保留運算緩衝區，大約需要 1GB 左右的保底空間。

結論：若要追求最高速的「純 GPU 推理」，你的 VRAM 必須大於「模型大小 + 至少 2GB ~ 3GB 的餘裕」。

🤝 進階玩法：CPU + GPU 混搭協同作戰 (50/50 分工)

不同於 vLLM 等伺服器框架遇到顯存不足會直接崩潰（OOM），Ollama 非常聰明，它具備 CPU Offloading（卸載） 的能力。如果我們把這個機制當作「主動策略」，就能做到跨級距的部署。

情境：用 12GB 顯卡，跑 24GB 的大模型（如 Qwen2.5-32B 或 Phi-3.5-MoE）

運作機制： 假設模型有 64 層，Ollama 評估你的 12GB VRAM 只能塞下 32 層，它就會自動把一半的層數放在 GPU，另一半放在電腦的系統記憶體 (CPU RAM) 裡。這就是所謂的 50% GPU + 50% CPU 混搭。

優點（能力升級）： 讓你能在消費級電腦上，執行原本需要高階伺服器才能跑的 30B+ 級別模型，獲取更強大的邏輯推理、程式碼編寫與資安分析能力。

代價（降速求生）： 系統記憶體 (DDR4/DDR5) 的頻寬遠低於 GPU VRAM，且資料必須頻繁跨越 PCIe 匯流排。這會導致模型生成速度從原本順暢的 30~40 tok/s，下降到 5~10 tok/s 左右。

先決條件： 你的電腦必須要有足夠大的系統記憶體（建議 32GB 甚至 64GB RAM），否則連 CPU 混搭都會失敗。

🎯 實戰：三張主流顯卡的「真實定位」

根據上述鐵則與混搭策略，我們來盤點這三張卡的真實戰力：

1. RTX 3060 Ti (8GB VRAM) —— 被 8GB 封印的算力

純 GPU 甜蜜點： Llama-3.1-8B (Q4_K_M) 或 Gemma-2-9B (Q4_K_M)。能以 40+ tok/s 極速運行。

混搭極限： 如果硬跑 14B 模型 (約 8.5GB)，Ollama 會把少量運算丟給 CPU。雖然能跑，但速度會掉到 5~8 tok/s，體驗不佳。

2. RTX 3060 (12GB VRAM) —— 窮人 LLM 神卡

硬體特性： 多出來的 4GB VRAM 讓它成為平民神器。

純 GPU 甜蜜點： Qwen-2.5-14B (Q4_K_M)。14B 模型 (8.5GB) + KV Cache + Overhead 剛好能 100% 塞進 12GB VRAM 裡，穩穩跑出約 15~20 tok/s 的速度。

混搭極限 (50/50)： 配合 32GB 系統 RAM，它可以勉強扛起 Gemma-4-27B 或 32B 等級模型，速度約落在 3~6 tok/s，適合用在不在乎速度、只求準確度的批次處理任務。

3. RTX 4070 (12GB VRAM) —— 速度王者，策略彈性高

硬體特性： 算力極強，GDDR6X 頻寬達 504 GB/s。

純 GPU 甜蜜點： 同樣最適合 14B 級別模型，但速度遠超 3060，可達 30~40+ tok/s，幾乎是即時串流。

混搭極限 (50/50)： 當它去跑 32B 等級模型（如 Qwen2.5-32B）時，雖然有一半層數在 CPU，但得益於 4070 本身強悍的算力與高頻寬，混搭速度有機會維持在 8~12 tok/s。這達到了「人類勉強可順暢閱讀」的及格線，是這張卡最具性價比的進階玩法。

💡 總結

在 Ollama 的世界裡，配置硬體其實是在做一道選擇題：

要極致速度（Streaming / 聊天對話）： 模型權重不要超過 GPU 總顯存的 70% ~ 75%（確保 100% GPU 執行）。

要極致智商（批次處理 / 複雜邏輯推演）： 善用 CPU + GPU 混搭，即便只有 12G 顯卡，只要配上 32GB 系統 RAM，也能讓模型智商翻倍，用 5~10 tok/s 的速度換取更精準的答案。
