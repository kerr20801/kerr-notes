# 🧠 AI & Machine Learning Cheat Sheet 

這是一份從「工程落地」與「解決問題」視角出發的 AI 技術選型指南。不談複雜的數學推導，只專注於：**這個演算法是什麼？它解決什麼問題？什麼時候該用它？**

整份指南分為三個維度：
1. **傳統機器學習**：處理結構化數據（Log、資料庫、統計指標）的極速首選。
2. **深度學習**：處理非結構化數據（影像、聲音、長篇文本）的重型武器。
3. **現代 AI 工程**：大語言模型（LLM）時代的實務落地架構。

---

## 第一部分：傳統機器學習 (Traditional Machine Learning)

💡 **核心特徵**：處理結構化表格數據的首選。依賴工程師手動設計特徵（Feature Engineering），運算資源需求低（大多 CPU 即可），推論極快，且具備高可解釋性。

### 1. 迴歸 (Regression) - 預測連續數值
* **線性迴歸 (Linear Regression)**
  * **適合情境**：預測房價、銷售額預估、評估廣告投入與營收的關係。適合呈現單純的線性趨勢。
* **多項式迴歸 (Polynomial Regression)**
  * **適合情境**：預測非線性成長的數據，例如傳染病擴散速度、特定物理資源（如 Disk IO）的消耗曲線。
* **正規化迴歸 (Ridge / Lasso Regression)**
  * **適合情境**：當特徵（欄位）極多且彼此高度相關時，用來防止模型過度擬合（Overfitting）。Lasso 還具備自動篩選重要特徵的能力。

### 2. 分類 (Classification) - 預測離散類別
* **邏輯迴歸 (Logistic Regression)**
  * **適合情境**：最基礎的二元分類。例如：垃圾郵件分類（是/否）、單一 API 請求是否異常。
* **單純貝氏 (Naive Bayes)**
  * **適合情境**：資料量小且多為文字特徵時。例如：早期的新聞文章自動標籤分類、簡單的惡意 Payload 關鍵字識別。
* **支持向量機 (SVM)**
  * **適合情境**：樣本數不多但特徵空間很複雜的情況。例如：高維度的邊界偵測、簡單的生物基因分類。

### 3. 強大樹狀模型 (Tree-based Models) - Kaggle 競賽常勝軍
* **隨機森林 (Random Forest)**
  * **適合情境**：顧客流失預測、銀行貸款審核。適合處理包含類別與數值的混合資料，抗雜訊能力強，不容易看錯特徵。
* **XGBoost / LightGBM**
  * **適合情境**：處理結構化表格數據的霸主。適合電商點擊率（CTR）預測、使用者行為精準分類。LightGBM 在極大資料量下擁有壓倒性的速度優勢。
* **CatBoost**
  * **適合情境**：與上述同屬 Gradient Boosting 家族，但對「類別型特徵（Categorical Features）」有極致優化（不需手動做 One-Hot Encoding）。適合處理含大量文字標籤的表格資料，如推薦系統、資安維度分類。

### 4. 非監督式學習 (Unsupervised Learning) - 無標籤找規律
* **K-Means 分群**
  * **適合情境**：客戶特徵細分（客群分析）、探索性資料分析（EDA），把行為相似的人群或 IP 群聚出來。
* **DBSCAN 分群**
  * **適合情境**：地理座標資料分析（如外送熱區、異常存取地預測）。能自動識別不規則形狀的群集，並直接排除離群雜訊。

### 5. 異常檢測 (Anomaly Detection) - 尋找極端值與雜訊
* **z-score (標準分數)**
  * **適合情境**：最基礎的統計過濾。當資料符合常態分佈時，快速剔除超過特定標準差的極端數值（例如：伺服器 CPU 瞬間飆高、UPS 電池劣化數值）。
* **孤立森林 (Isolation Forest, IF)**
  * **適合情境**：非常適合處理高維度資料。透過隨機切分來「孤立」樣本，異常點通常最快被切出。常用於：信用卡盜刷偵測、應用層防火牆（WAF）的異常流量過濾。
* **ECOD (Empirical Cumulative Distribution Functions for Outlier Detection)**
  * **適合情境**：運算極快且無需假設資料分佈（無參數模型）。適合多指標（Multi-metric）的系統健康度監控、跨維度 IP 威脅分析，不怕維度災難。

### 6. 降維 (Dimensionality Reduction) - 精簡特徵、視覺化
* **主成分分析 (PCA)**
  * **適合情境**：預處理階段的資料壓縮，保留最大變異數；或消除欄位之間的共疊（多元共線性）影響，加速後續模型訓練。
* **t-SNE / UMAP**
  * **適合情境**：將幾百個特徵的高維度數據，壓縮到 2D 或 3D 空間，畫成圖表來直觀觀察資料的聚集與分群趨勢。

### 7. 傳統時間序列與序列標記 (Time Series & Sequence Models)
* **Prophet (Meta 開源)**
  * **適合情境**：專門處理具備強烈週期性（年、週、日）和節假日效應的商業數據。例如：預測未來一年的電商業務量、預測伺服器連線數以進行自動擴容（Auto-scaling）。對缺失值容忍度高。
* **隱馬可夫模型 (HMM) & 條件隨機場 (CRF)**
  * **適合情境**：需要即時推論（CPU Only）且具備高可解釋性的序列任務。CRF 常搭配 HMM 用於安全事件序列標記、實體識別（NER），每一步觸發的特徵皆可追溯審計。

### 8. 相似度比對與資訊檢索 (Similarity & Information Retrieval)
* **BM25 (Best Matching 25)**
  * **適合情境**：不需要 GPU，基於詞頻與反向文件頻率的極致關鍵字比對。適合在海量歷史文本（如 Email 回信庫、客服紀錄）中找出最相關的舊資料。也是現代 RAG 架構最可靠的基石之一。

## 第二部分：深度學習 (Deep Learning)

💡 **核心特徵**：處理非結構化數據（影像、聲音、長篇自然語言）的首選。模型能夠「自動萃取特徵」，但極度仰賴海量資料與強大的運算資源（GPU），且過程多為黑箱（Black-box）。

### 1. 電腦視覺 (Computer Vision)
* **卷積神經網路 (CNN 及變體如 ResNet, YOLO)**
  * **適合情境**：所有與影像、影片相關的任務。例如：工廠產線的瑕疵檢測、自動駕駛的物件識別、醫學 CT 影像診斷、人臉辨識。

### 2. 序列與時間模型 (Sequential Models)
* **長短期記憶網路 (LSTM / GRU)**
  * **適合情境**：與時間順序、前後文具強烈相關性的資料。內建「記憶機制」，例如：早期的語音辨識、複雜機具的震動頻率異常預測、氣象預報。

### 3. 基礎模型與自然語言 (Foundation Models & NLP)
* **Transformer (及變體如 BERT, GPT 系列, Llama)**
  * **適合情境**：現代 AI 的核心架構。採用「自注意力機制（Self-Attention）」，專精於機器翻譯、長文本摘要、程式碼自動補全。目前也延伸至影像處理（Vision Transformer, ViT）與多模態任務。

### 4. 向量檢索與特徵表達 (Embeddings)
* **嵌入模型 (Embedding Models)**
  * **適合情境**：將文字、圖片壓縮轉換成多維數學向量空間。透過計算距離（如餘弦相似度）來判斷關聯性。專解「沒有相同關鍵字，但語意相近」的模糊搜尋任務。

### 5. 異常生成與重建 (Reconstruction-based Detection)
* **自動編碼器 (Autoencoder, AE / VAE)**
  * **適合情境**：進階的非监督式異常偵測。讓模型學習將「正常資料」壓縮再還原，當「異常資料」輸入時，模型會因為無法完美還原（產生極大重建誤差）而將其抓出。

## 第三部分：現代 AI 工程實務 (AI Engineering & LLMOps)

💡 **核心特徵**：在生成式 AI（GenAI）時代，焦點從「從頭訓練模型」轉移到「串接、微調、部署與系統整合」。

### 1. 大型語言模型應用架構
* **RAG (Retrieval-Augmented Generation / 檢索增強生成)**
  * **解決什麼痛點**：LLM 會產生幻覺，且不知道公司私有/最新的資料。
  * **實務作法**：先透過 BM25 或向量資料庫（Vector DB）檢索出相關內部文件，再將文件作為 Context 餵給 LLM 讓它基於事實回答。
* **Agentic Workflow (AI 代理工作流 / Function Calling)**
  * **解決什麼痛點**：LLM 只能「說話」，不能「做事」。
  * **實務作法**：賦予 LLM 呼叫外部 API 的能力。例如：讓 AI 在分析完 Log 發現攻擊後，自主決定呼叫防火牆 API 封鎖 IP，打造 Chat-to-Ops 自動化體系。

### 2. 模型微調與對齊 (Fine-Tuning & Alignment)
* **PEFT / LoRA (低秩微調)**
  * **解決什麼痛點**：重新訓練 LLM 成本太高。
  * **實務作法**：凍結大模型主體參數，只訓練旁支的極小模組。只需單張 GPU 即可讓開源模型學會公司的特定 Log 格式或專業領域術語。
* **RLHF (基於人類回饋的強化學習)**
  * **解決什麼痛點**：大模型講話不討喜或帶有危險性。
  * **實務作法**：藉由人類對模型輸出的評分，引導模型學會「有禮貌、安全、符合人類預期」的對話方式（AI Alignment）。

### 3. 部署與推論加速 (Deployment & Inference)
* **模型量化 (Quantization - GGUF / AWQ)**
  * **實務作法**：將模型權重從高精度（16-bit）壓縮至低精度（8-bit 或 4-bit），稍微犧牲精度，換取體積大幅縮小。實現在消費級 GPU 或純 CPU 伺服器上運行大型本地模型（Local LLM）。
* **高效能推論引擎 (vLLM / Ollama)**
  * **實務作法**：專為大模型設計的 Serving 框架。透過 PagedAttention 等底層記憶體優化技術，解決併發請求（Concurrency）時的效能瓶頸，提供企業級的 API 服務。
